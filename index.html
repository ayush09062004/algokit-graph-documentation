AlgoKit Graph · API User Guide

Illustrated guide to every AlgoKit Graph API

AlgoKit Graph is a C++17 graph algorithms library with Python bindings. This guide walks through every public method on Graph, every result type, and DisjointSet — each with its signature, parameters, return value, a worked diagram, and runnable Python.

Getting started

Every graph is created through one of two factory methods, then built up with add_edge. Vertices are always integers 0 .. n-1, fixed at construction time.

graph = algokit.Graph.directed(num_vertices)
Creates a directed graph with a fixed vertex count. Edges are one-directional.
import algokit

# 6 vertices, numbered 0..5
graph = algokit.Graph.directed(6)
graph.add_edge(0, 1)          # weight defaults to 1.0
graph.add_edge(1, 2, 2.5)     # explicit weight

print(graph)                  # Graph(num_vertices=6, num_edges=2, directed=True)
graph = algokit.Graph.undirected(num_vertices)
Creates an undirected graph. Internally, add_edge(u, v) registers the edge in both directions’ adjacency lists automatically.
graph = algokit.Graph.undirected(5)
graph.add_edge(0, 1, 4.0)
graph.neighbors(1)   # -> also contains an Edge back to 0, even though you only called add_edge once
Vertex ids outside [0, num_vertices) raise IndexError (mapped from C++ std::out_of_range) on any method that touches a vertex.
01

Building & inspecting a graph

Methods for adding edges in bulk, reading the graph back out, and basic metadata.

graph.add_edge(from_vertex, to_vertex, weight=1.0)
mutates graph
Adds one edge. On an undirected graph this updates both endpoints’ adjacency lists.
ParamTypeNotes
from_vertexintSource vertex.
to_vertexintDestination vertex.
weightfloatOptional, defaults to 1.0.
graph.add_edges(edges)
mutates graph
Bulk-adds edges from an iterable of 2- or 3-tuples: (from, to) or (from, to, weight). Equivalent to calling add_edge in a loop, but avoids the Python call overhead per edge.
graph = algokit.Graph.directed(4)
graph.add_edges([
    (0, 1),        # weight -> 1.0
    (1, 2, 3.5),
    (2, 3, 1.0),
])
graph.add_adjacency_matrix(adjacency_matrix)
mutates graph
Builds edges from an n×n matrix. Any non-zero cell matrix[i][j] (including negative values) becomes an edge i → j with that weight. For undirected graphs only the upper triangle is read (the matrix is assumed symmetric), so each edge is added once, not twice.
graph = algokit.Graph.undirected(3)
graph.add_adjacency_matrix([
    [0, 2, 0],
    [2, 0, 5],
    [0, 5, 0],
])
graph.edge_count()   # -> 2
Watch out: a matrix cell equal to exactly 0 is treated as “no edge”, so a genuine zero-weight edge cannot be expressed this way — use add_edge(i, j, 0.0) instead.
graph.vertex_count()
returnsint — fixed at construction.
graph.edge_count()
returnsint — number of add_edge calls (each undirected edge counts once).
graph.is_directed()
returnsbool
graph.neighbors(vertex)
returnslist[Edge] — outgoing edges from vertex.
graph.edges()
Returns list[GraphEdge] — every edge as it was added, each with explicit from_vertex/to_vertex/weight. For undirected graphs this list has one entry per edge (not two) — it mirrors add_edge calls, not the adjacency list.
for e in graph.edges():
    print(e.from_vertex, "->", e.to_vertex, "weight", e.weight)
graph.transpose()
directedraises RuntimeError on undirected
Returns a new Graph with every edge reversed. Used internally by strongly_connected_components() (Kosaraju’s algorithm), and generally useful for any “who points to me?” query.
g = algokit.Graph.directed(3)
g.add_edges([(0, 1), (1, 2), (0, 2)])
gt = g.transpose()
[(e.from_vertex, e.to_vertex) for e in gt.edges()]   # -> [(1,0), (2,1), (2,0)]

Traversal — BFS & DFS

Both traversals work on directed and undirected graphs and start from a single source vertex; unreachable vertices are simply omitted from the result.

result = graph.bfs(source)
BFSResult
Breadth-first search from source. Distances count edges (hops), not weights — use dijkstra/bellman_ford if you need weighted distance.
BFSResult methodReturns
order()list[int]
distance()list[int]
parent()list[int] (-1 = no parent / unreached)
result = graph.bfs(0)
result.order()        # visit order, e.g. [0, 1, 2, 3, 4]
result.distance()     # hop count from source, indexed by vertex id
result.parent()       # BFS-tree parent of each vertex, -1 for the source / unreached
Time: O(V + E)
result = graph.dfs(source)
DFSResult
Recursive depth-first search from source, tracking discovery and finish times (classic CLRS-style timestamps) — handy for cycle/topological-sort style reasoning of your own.
DFSResult methodReturns
order()list[int]
parent()list[int]
discovery_time()list[int]
finish_time()list[int]
result = graph.dfs(0)
result.discovery_time()[3], result.finish_time()[3]
Time: O(V + E) · recursive — very deep graphs can hit Python/C++ stack limits

Cycle detection

Two separate boolean checks — one per graph type. Calling the wrong one for your graph’s type raises an error rather than silently answering the wrong question.

graph.has_cycle()
undirectedraises RuntimeError on directedbool
DFS with parent-tracking: a back edge to any already-visited vertex other than the immediate parent means a cycle.
algokit.Graph.undirected(4)  # add a triangle
g.has_cycle()   # -> True
Time: O(V + E)
graph.has_directed_cycle()
directedraises RuntimeError on undirectedbool
Three-colour DFS (white / gray / black). A gray-to-gray edge is a back edge → cycle.
g = algokit.Graph.directed(4)
g.add_edges([(0,1), (1,2), (2,3), (3,1)])
g.has_directed_cycle()   # -> True  (1 -> 2 -> 3 -> 1)
Time: O(V + E)
05

DAG — topological ordering

Two independent implementations of the same result, useful for cross-checking or for picking the style you prefer. Both require a directed acyclic graph.

result = graph.topological_sort()
directedraises RuntimeError if not a DAG
DFS-based: runs a full DFS, then reverses the post-order (finish-time) sequence. Internally validates via has_directed_cycle() first.
result = graph.topological_sort()
result.order()   # e.g. [0, 2, 1, 4, 3, 5]
Time: O(V + E)
result = graph.kahn_topological_sort()
directedraises RuntimeError if not a DAG
Kahn’s algorithm: repeatedly emits zero-indegree vertices. If the produced order is shorter than vertex_count(), the leftover vertices form a cycle and an error is raised.
result = graph.kahn_topological_sort()
result.order()
Time: O(V + E)

Shortest paths

Four algorithms, each suited to a different weight profile. All single-source variants return a ShortestPathResult; Floyd-Warshall returns all pairs.

result = graph.shortest_path(source)
unweighted (BFS)
Counts edges, not weight — every edge is treated as cost 1 regardless of the weight you gave it. Use this when you only care about hop count on an unweighted (or weight-irrelevant) graph; use dijkstra or bellman_ford if the actual edge weights matter.
result = graph.shortest_path(0)
result.distance(4)     # hop count, ignores real edge weights
result.path_to(4)      # list[int], fewest-edges path
Time: O(V + E)
result = graph.dijkstra(source)
non-negative weights
Binary-heap Dijkstra with lazy deletion of stale queue entries. Requires non-negative edge weights — it does not check for negative edges, so a negative weight will silently produce an incorrect (too-optimistic) distance rather than raising an error.
ShortestPathResult methodReturns
distance(vertex)float — inf if unreachable
parent(vertex)int — -1 if none
is_reachable(vertex)bool
path_to(vertex)list[int] — empty if unreachable
result = graph.dijkstra(0)
result.distance(4)     # weighted shortest distance
result.path_to(4)      # [0, 1, 3, 4]
Time: O(E log V)
result = graph.bellman_ford(source)
negative weights okraises RuntimeError on negative cycle
Relaxes every edge up to V-1 times (with early-exit once nothing changes), then does one more pass to detect a reachable negative-weight cycle.
result = graph.bellman_ford(0)
result.distance(4)
try:
    graph.bellman_ford(0)
except RuntimeError as e:
    print(e)   # "...detected a reachable negative-weight cycle."
Time: O(V · E)
Known limitation: on an undirected graph, Bellman-Ford here only relaxes each edge in the direction it was added to the edge list, not both directions. In practice this can report some vertices as unreachable even though they are connected. For undirected graphs, prefer dijkstra() (non-negative weights) or restructure the input as two directed edges per undirected edge before calling bellman_ford.
result = graph.floyd_warshall()
all pairsraises RuntimeError on negative cycle
Classic O(V³) DP over all pairs; correctly symmetrizes edges for undirected graphs. Also reconstructs paths via a next matrix.
FloydWarshallResult methodReturns
distance(from_vertex, to_vertex)float
is_reachable(from_vertex, to_vertex)bool
path(from_vertex, to_vertex)list[int]
result = graph.floyd_warshall()
result.distance(2, 4)
result.path(2, 4)
Time: O(V³) · Space: O(V²)

Minimum spanning tree

Both algorithms operate on undirected graphs and produce the same total weight for a connected graph; Prim additionally handles disconnected graphs by building a spanning forest.

result = graph.kruskal()
undirectedraises RuntimeError on directed
Sorts all edges by weight, then adds each one that connects two different components, tracked with a DisjointSet.
result = graph.kruskal()
result.total_weight()
result.edges()          # list[GraphEdge] in the MST
Time: O(E log E)
result = graph.prim()
undirectedraises RuntimeError on directed
Grows the tree one vertex at a time from a min-priority queue of frontier edges. Restarts from any unvisited vertex to also cover disconnected graphs (producing a minimum spanning forest).
result = graph.prim()
result.total_weight()
result.edge_count()
Time: O(E log V)

DisjointSet (Union–Find)

A standalone data structure — not tied to a Graph instance — with path compression and union by size. Used internally by kruskal(), and exposed directly for your own algorithms.

dsu = algokit.DisjointSet(n)
Creates n singleton sets, elements 0 .. n-1.
MethodReturnsNotes
find(x)intRoot of x’s set, with path compression.
unite(x, y)boolTrue if the sets were merged, False if already unioned.
connected(x, y)bool
component_size(x)intSize of the set containing x.
component_count()intTotal number of sets remaining.
dsu = algokit.DisjointSet(5)
dsu.unite(0, 1)
dsu.unite(2, 3)
dsu.unite(1, 2)
dsu.connected(0, 3)        # -> True
dsu.component_count()      # -> 2   ({0,1,2,3}, {4})
Amortized: O(α(N)) per operation (inverse Ackermann — effectively constant)

Edge & GraphEdge

Two small read-only value types you get back from neighbors() and edges() respectively — the difference is whether the source vertex is implicit or explicit.

Edge
Returned by graph.neighbors(v) — the source is implicit (it’s the vertex you queried).

weightfloat
GraphEdge
Returned by graph.edges() and MST results — carries the source explicitly.
